# plotnote

이 프로젝트에 대해 소개할게.

[책장(Bookshelf)]
프로젝트들을 볼 수 있음. 카드형식으로 프로젝트 이미지 썸네일로 뜸.
최근 수정 기준으로 뜨지만 즐겨찾기를 해둔 프로젝트가 더 최우선으로 뜸.
공유책장엔 공유된 프로젝트만 뜸. 내 책장엔 내 책장 프로젝트 전부가 뜸.

[프로젝트(Project)]
일정규격으로 표시되는 이미지썸네일, 그 밑으로 프로젝트 명이 있는 카드가 그리드로 나열되어 있음.
최근 수정 기준으로 뜨지만 즐겨찾기를 해둔 프로젝트가 더 최우선으로 뜸.
선택시 해당 프로젝트에 대한 이름, 이미지, 설정, 메모 등을 적을 수 있는 화면이 뜸.
추가하는 방식으로 에피소드별 간략한 메모를 적을 수도 있음.
에피소드 관련 메모는 추가하는 식으로 적혀서 각각의 메모를 드래그로 순서 조정하여 에피소드 메모의 순서를 변경할 수 있음.
이 에피소드 메모는 에피소드 관리 화면에서 작성하는 메모랑 같은거임(연동).
추가한 만큼 에피소드가 자동으로 추가됨.
에피소드를 선택함으로써 해당 에피소드 페이지 관리 화면으로 갈 수도 있고, 따로 어딘가를 선택하면 해당 프로젝트의 에피소드 관리 화면으로 갈 수도 있음.
해당 프로젝트에 대한 설정도 할 수 있음(에피소드 생성시의 기본 페이지수 설정, 시작 페이지 홀수/짝수 페이지로 설정, 왼오 읽기 오왼 읽기 등등...)

[에피소드(Episode)]
일정규격으로 표시되는 이미지썸네일, 그 옆으로 에피소드명과 그 밑으로 에피소드 메모(검정색 굵은 글씨? 아무튼 차이가 있음)가 보임.
카드배열식이 아니라 리스트 식임.
오른쪽 위엔 에피소드 추가 버튼이 있음.
드래그로 에피소드 위치 변경이 가능함. 해당 에피소드를 누르면 해당 에피소드에 대한 세부메모를 적을 수 있는 창이 뜸.
추가하는 식으로(또는 프로젝트 설정에 따라 기본적으로 생성된 페이지 수에) 페이지별 메모도 작성할 수 있고 이 페이지 메모는 페이지 관리 화면에서 작성하는 메모랑 같은거임.(연동)
해당 에피소드만 기본페이지수 설정, 시작페이지 홀수짝수 설정 등 프로젝트설정과 다른 사항이 있으면 개별 수정이 가능함.

[페이지(Page)]
에피소드 설정에 따라 단면 또는 양면으로 페이지가 표시됨.
양면기준 3개마다 행이 나뉘고 단면기준 6개...또는 5개...마다 행이 나뉨.
ctrl+스크롤 시 보여지는 갯수가 달라짐. 맨 끝에 페이지 추가 버튼이 있음.
드래그로 페이지 순서 변경이 가능함.
컷 분할은 페이지 화면에선 불가능함. 메모 또는 컷이동은 가능하마
페이지 썸네일 상단에 페이지 메모가 작은 글씨로 적혀있고(최대 2줄, 넘어갈 시 ...으로 표시) 클릭시 해당 페이지의(단면일 경우 단면, 양면일경우 양면) 페이지메모(검정색 굵은글씨) 가 있고 세부메모를 작성할 수 있음. 
추가하는 식으로 개별 컷별 메모를 작성할 수 있는데 이 컷별 메모는 컷 관리 화면에서 드래그 식으로 끌어와 나눠둔 컷에 집어넣을 수 있음. 
기본 24페이지 (차후 에피소드 추가 시점에 페이지 수를 조정하거나, 특정 프로젝트 내에서는 에피소드 생성시 기본 n페이지로 만들도록 설정하거나, 만들어진 에피소드 내에서 페이지를 추가삭제 가능하게 할 예정)
페이지 화면에서도 컷이 보이긴 보여서, 컷 안의 글씨도 작은글씨지만 보이긴 보임. 너무 길면 ...로 표시되고.

[컷(cut)]
에피소드 설정에 따라 단면 또는 양면으로 페이지가 표시됨.
기본 전체 페이지가 보이도록 화면이 표시되고, ctrl+스크롤로 마우스 기준 확대 축소가 가능하며 왼쪽아래건 오른쪽아래건 오른쪽이건 왼쪽이건에 전체 페이지가 보이는 화면 표시로 되돌리는 버튼이 있음.
기본적으로 초기 전체 컷이 하나가 표시되어 있으며(데이터로는 존재하지 않음), 아까 그 전체화면 표시 버튼쪽에 컷 수정 버튼이 있어서 해당 버튼을 active 시킨 상태로 컷에서 특정 방향으로 드래그 하면 해당 방향 모양으로 컷이 나뉨.
이런 식으로 컷이 분할됨.
컷별 메모를 드래그앤 드롭으로 컷에 지정할 수 있음, 컷별 세부메모를 작성할 수 있음. 나중엔 프로젝트 별로 등장인물이나 장소 등을 설정해서 해당 설정을 불러와서 해당 인물의 대사라던가, 특정 장소임을 나타낼 수도 있음)

현재:
LocalStorage 기반, 에피소드 단위 설계

차후:
로그인 기능 추가
서버 저장
책장 단위 관리
다중 계정 공유 (readonly, commentonly, editor, owner로 권한이 나뉨)
에피소드 단위 구조는 유지, 이건 무료 사용자 용

따라서:
프론트 상태 구조는 서버 JSON 구조와 1:1 대응 가능해야 함
임시 데이터 구조, index 기반 접근 금지

설계 철학 (중요)
index를 key로 사용하지 않는다.
display용 번호와 내부 식별 ID를 분리한다.
상태 변경은 항상 불변성을 유지한다.
“지금은 안 쓰지만 나중에 필요할 필드”는 미리 설계하되, UI 구현은 하지 않는다.
기능 구현보다 데이터 모델 안정성을 우선한다.

이 프로젝트는 지금 당장 잘 돌아가는 것보다 2년 뒤에도 안 부서지는 구조가 핵심.
