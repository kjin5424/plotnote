# plotnote 화면 디자인 설명

# **1. 전체 레이아웃**

- 화면은 크게 **사이드바(좌측)**와 **메인 콘텐츠 영역(우측)**의 2단 구조로 구성됩니다.
- **사이드바**는 접거나 펼칠 수 있어 사용자가 작업 공간을 넓게 활용할 수 있습니다.

---

# **2. 사이드바 (Sidebar)**

- **기능:**
  - **로고 및 사이드바 접기/펼치기 버튼:** 최상단에 위치합니다.
  - **검색창:** 사이드바 상단에 있어 전체 프로젝트, 에피소드, 페이지, 컷 내에서 키워드 검색을 실행할 수 있습니다. 검색창을 클릭하면 **검색 모달**이 화면 중앙에 나타나며, 사용자는 여기서 원하는 내용을 검색할 수 있습니다. 각각 프로젝트, 에피소드, 페이지, 컷 별로 검색 결과를 볼 수 있습니다.
  - **트리 메뉴:** 각 책장의 프로젝트들을 계층 구조로 보여줍니다. (계층 설명 : 책장 > 프로젝트 > 에피소드 > 페이지 ( > 컷 ) ) (단, 컷은 펼친 트리 메뉴에서는 보이지 않고, 접힌 상태에서만 아이콘으로 표시됩니다.
  - **회원 정보:** 회원 아이콘과 닉네임이 보입니다.
  - **접힌 사이드바:** 로고, 현재 위치한 책장, 프로젝트, 에피소드, 페이지, 컷의 아이콘, 회원 아이콘. 아이콘 위에 마우스를 올려놓으면 하위 에셋 리스트가 표시됩니다.(예: 에피소드 아이콘 위에 마우스를 올리면 페이지 리스트가 툴바 형태로 표시) 특정 행을 클릭 시 해당 페이지로 이동됩니다.
- **트리 메뉴 구성:**
  - **즐겨찾기:** 즐겨찾기 지정한 프로젝트들을 모아보는 공간입니다.
  - **나의 책장:** 내가 생성한 모든 프로젝트가 나열됩니다.
  - **공유 책장:** 다른 사용자로부터 공유받은 프로젝트들이 나열됩니다.
  - 각 그룹(즐겨찾기, 나의 책장 등)은 접거나 펼칠 수 있으며, 내부에 속한 프로젝트 목록을 보여줍니다.

---

# **3. 메인 콘텐츠 영역 (Workspace)**

- 화면은 크게 **상단 헤더**와 **콘텐츠 본문**으로 구성되어 있습니다.
- 화면에 따라 **푸터**와 콘텐츠 본문 내부의 **우측 사이드바**가 있습니다.

---

## **3-1. Bookshelf 화면**

- **상단 헤더:**
  - 현재 화면의 이름인 "**책장**"이 표시됩니다.
  - 우측에는 "**+ 새 프로젝트**" 버튼이 있어 클릭 시 새 프로젝트 생성 모달이 나타납니다.
- **콘텐츠 본문:**
  - 사이드바의 그룹과 동일하게 "**즐겨찾기**", "**나의 책장**", "**공유 책장**" 섹션으로 구분됩니다.
  - 각 섹션은 헤더(아이콘, 섹션 이름, 프로젝트 개수)를 가지며, 클릭하여 해당 섹션의 프로젝트 목록을 접거나 펼칠 수 있습니다.
  - **프로젝트 목록:**
    - 각 섹션 내의 프로젝트들은 **카드 형태**의 그리드(grid) 뷰로 표시됩니다.
    - **프로젝트 카드 구성:**
      - **썸네일 이미지:** 카드의 상단 영역을 차지하며, 프로젝트를 시각적으로 대표합니다.
      - **프로젝트 정보:**
        - **제목:** 프로젝트의 이름입니다.
        - **메타데이터:** 총 페이지 수, 마지막 수정일 등의 정보가 표시됩니다.
        - **참여자:** 프로젝트에 참여한 멤버들의 이름과 역할(소유자, 편집자 등)이 뱃지 형태로 표시됩니다.
      - **즐겨찾기 버튼(북마크 아이콘):** 카드 인포 영역의 우측 상단에 위치하며, 클릭하여 즐겨찾기에 추가하거나 해제할 수 있습니다.
    - '나의 책장' 섹션에는 마지막에 '**+ 새 프로젝트**' 카드가 있어, 이를 통해서도 새 프로젝트를 생성할 수 있습니다.
- **핵심 상호작용 (Interaction)**
  - **새 프로젝트 생성:**
    1. 메인 콘텐츠 헤더의 `+ 새 프로젝트` 버튼 또는 '나의 책장' 섹션의 `+ 새 프로젝트` 카드를 클릭합니다.
    2. **'새 프로젝트' 모달**이 나타납니다.
    3. 모달 안에서 프로젝트 **타이틀**을 입력하고, **기본 페이지 수, 읽기 방향(좌->우(LTR)/우->좌(RTL)), 기본 뷰(단면/양면), (기본 뷰가 양면일 시 시작페이지 홀수/짝수)** 등의 초기 설정을 지정한 후 '만들기' 버튼을 눌러 프로젝트를 생성합니다.
  - **프로젝트 접근:** 사용자가 프로젝트 카드를 클릭하면 해당 **Project 관리 화면**으로 이동합니다.

## 3-2. Project 화면

- **상단 헤더:**
  - 좌측에는 현재 작업 중인 **프로젝트의 제목**(예: "고양이와 강아지")이 표시됩니다.
  - 프로젝트 제목 옆에는 **즐겨찾기 버튼(북마크 아이콘)**이 있어 이 프로젝트를 즐겨찾기에 추가하거나 해제할 수 있습니다.
  - 우측에는 여러 기능 버튼이 위치합니다.
    - **`+ 에피소드 추가`**: 새 에피소드를 생성하는 버튼입니다.
    - **`메모 패널` (가로점 3개 아이콘)**: 화면 우측에 현재 열려있는 프로젝트 관련 메모(프로젝트>에피소드>페이지>컷)를 확인할 수 있는 플로팅 패널을 열거나 닫습니다.
- **콘텐츠 1 - 프로젝트 정보:**
  - **프로젝트 메모:** 프로젝트의 전체적인 줄거리나 핵심 내용을 기록하는 텍스트 영역입니다. 작성, 수정가능합니다.
  - **프로젝트 세부:** 장르, 대상 독자, 캐릭터 설정, 연재 주기 등 구조화된 세부 정보를 기록하는 텍스트 영역입니다. 작성, 추가, 수정, 삭제 가능합니다.
    - 예시
      - #장르 | SF
      - #캐릭터 | 주인공, 이름 홍길동, 나이 18
      - #복선 | 아버지가 사실 진짜 친아버지이다
  - **프로젝트 정보:** 프로젝트에 포함된 총 에피소드 수, 전체 페이지 수, 마지막 작업일 등의 통계 데이터를 보여줍니다. 프로젝트 공유 버튼이 있으며, 클릭 시 현재 공유 중인 멤버 리스트를 확인할 수 있고(편집자, 댓글 작성자, 읽기 허용 등) 링크 공유를 통해 다른 회원에게 해당 프로젝트를 공유할 수 있습니다.
  - **프로젝트 설정:** 프로젝트의 기본 페이지 수, 읽기 방향, 기본 뷰, 시작 페이지와 같은 전반적인 설정을 확인하고 변경할 수 있습니다.
- **콘텐츠 2 - 에피소드 목록:**
  - "**에피소드 목록**"이라는 제목과 함께, 해당 프로젝트에 속한 모든 에피소드가 **행(row) 기반의 리스트**로 표시됩니다.
  - 리스트 상단에는 전체 에피소드 개수와 **정렬 기능**(예: 최신 작업순, 에피소드 순)을 제공하는 드롭다운 메뉴가 있습니다.
  - **에피소드 리스트 구성 (각 행):**
    - **드래그 핸들:** 각 행의 맨 앞에 위치하며, 사용자가 마우스로 클릭하여 에피소드의 순서를 자유롭게 변경할 수 있습니다.
    - **에피소드 번호 및 제목:** 에피소드의 고유 번호(예: `ep.1`)와 제목(예: "첫 만남")이 표시됩니다. 제목은 에피소드 리스트 하단의 에피소드 추가 버튼으로 추가 시 ‘새 에피소드’로 표시되며, 모든 제목은 수정 가능합니다.
    - **시놉시스:** 에피소드의 핵심 내용을 요약한 한 줄 메모를 작성, 수정할 수 있습니다.
    - **상태:** 해당 에피소드의 작업 진행 상태를 나타내는 뱃지(예: 예정, 진행중, 완료)가 표시됩니다.
    - **메타데이터:** 페이지 수, 마지막 수정일 등의 정보가 표시됩니다.
    - **더보기 메뉴(점 3개 아이콘):** 행의 맨 끝에 위치하며, 클릭 시 '에피소드 설정 변경', '삭제' 등의 추가 메뉴가 나타납니다.
- **핵심 상호작용 (Interaction)**
  - **새 에피소드 생성:**
    1. 상단 헤더의 `+ 에피소드 추가` 버튼 또는 에피소드 리스트 최하단의 `+ 에피소드 추가` 리스트를 클릭합니다.
    2. 에피소드 리스트의 맨 아래에 새로운 행이 추가되면서, 제목 등을 바로 입력할 수 있는 상태가 됩니다. 또는 별도의 '새 에피소드' 모달이 나타날 수 있습니다.
  - **에피소드 순서 변경:** 사용자가 각 에피소드 행 앞의 **드래그 핸들**을 잡고 위아래로 끌어다 놓으면 리스트 내에서 순서가 즉시 변경됩니다.
  - **에피소드 접근:** 사용자가 리스트에서 특정 에피소드 행을 **클릭**하면 해당 **Episode 관리 화면**으로 이동하여 페이지와 컷 단위의 상세 작업을 진행할 수 있습니다.

## 3-3. Episode 화면

- **상단 헤더:**
  - 좌측에는 현재 작업 중인 **프로젝트의 제목**(예: "고양이와 강아지")이 표시됩니다. `-에피소드 리스트` 라는 문구를 프로젝트의 제목 우측에 추가시킴으로써 Project 화면과의 차이점을 보입니다.
  - 프로젝트 메모를 확인할 수 있습니다. 에피소드 작업 시 프로젝트의 주요 내용을 복기할 수 있습니다.
  - 우측에는 여러 기능 버튼이 위치합니다.
    - **`+ 에피소드 추가`**: 새 에피소드를 생성하는 버튼입니다.
    - **`메모 패널` (가로점 3개 아이콘)**: 화면 우측에 현재 열려있는 프로젝트 관련 메모(프로젝트>에피소드>페이지>컷)를 확인할 수 있는 플로팅 패널을 열거나 닫습니다.
- **콘텐츠 1 - 에피소드 목록:**
  - 현재 프로젝트에 속한 모든 에피소드의 목록을 보여주는 ‘네비게이터’ 역할을 합니다.
  - 에피소드 목록을 상태별로("전체", "진행중", "완료", "예정") 필터링하여 각각의 탭으로 볼 수 있습니다.
  - **정렬 기능:** 목록의 순서를 변경할 수 있는 정렬 버튼이 있습니다.
  - **에피소드 리스트:**
    - Project 화면과 유사하게, 각 에피소드가 하나의 **행(row)**으로 표시됩니다.
    - 현재 작업 중인 에피소드 행은 하이라이트 처리되어 시각적으로 구분됩니다.
    - 각 행에는 썸네일, 에피소드 번호와 제목, 간단한 메모(작성, 수정 가능), 페이지 수, 그리고 작업 상태 뱃지가 포함됩니다.
  - **에피소드 추가 버튼:** 목록 하단에 `+ 에피소드 추가` 버튼이 있어, 이 화면을 벗어나지 않고도 새 에피소드를 생성할 수 있습니다.
- **콘텐츠 2 - 에피소드 상세:**
  - 선택된 에피소드의 내용을 구체적으로 확인하고 편집하는 주된 작업 공간입니다.
  - 수정 중인 에피소드의 **번호와 제목**(예: `ep.3` 이웃 친구들), 그리고 **총 페이지 수**가 표시됩니다.
  - 우측에는 에피소드의 **작업 상태**를 변경할 수 있는 드롭다운 메뉴(예: '● 완료')와 **개별 설정** (톱니바퀴 아이콘) 버튼이 있습니다.
  - 에피소드 작업 상태를 완료로 변경할 시, 미배정 컷 메모가 있는 경우 토스트 알림을 띄우며, 취소 또는 미배정 컷 메모 전체 삭제를 할 수 있습니다.
  - 상세 패널의 콘텐츠를 전환하는 탭이 있습니다. 기본적으로 에피소드의 글 정보를 관리하는 **'메모'** 탭이 활성화됩니다.
  - 탭의 오른쪽 끝에는 `+ 페이지 추가` 버튼이 있어, 현재 에피소드에 새 페이지를 추가할 수 있습니다.
  - 에피소드 공유 버튼이 있으며, 클릭 시 현재 공유 중인 멤버 리스트를 확인할 수 있고(편집자, 댓글 작성자, 읽기 허용 등) 링크 공유를 통해 다른 회원에게 해당 에피소드를 공유할 수 있습니다.
- **콘텐츠 3 - 에피소드 메모:**
  - **에피소드 전체 메모:**
    - **에피소드 메모:** 에피소드의 전반적인 시놉시스나 줄거리를 보여줍니다. 작성, 수정 가능합니다.
    - **에피소드 세부 메모:** 연출 방향, 분위기, 핵심 설정 등 구체적인 내용을 자유롭게 기록할 수 있는 텍스트 영역입니다. 작성, 추가, 수정, 삭제 가능합니다.
    - 예시
      - #복선 | 과거 회상씬에서 아버지로 유추되는 사람이 보임
      - #복선회수 | 출생신고서에 아버지가 친아버지로 적혀있다
      - #힐링편 | 다음 화가 심각하니까 그 전의 힐링편
  - **페이지별 메모 리스트:**
    - 에피소드를 구성하는 모든 페이지가 순서대로 나열됩니다. 페이지는 프로젝트 단계에서 설정한 페이지 수 또는 에피소드 별로 설정한 개별 설정 페이지 수 만큼 페이지 수가 있습니다.
    - 각 **페이지 행**은 **드래그 핸들, 페이지 번호(예: `p.01`), 한 줄 메모 입력 필드**로 구성됩니다.
    - 사용자는 이 리스트를 통해 전체 페이지의 흐름을 한눈에 파악하고, 각 페이지에 들어갈 핵심 연출이나 대사 아이디어를 빠르게 기록할 수 있습니다.
    - **드래그 앤 드롭**으로 페이지의 순서를 쉽게 변경하여 이야기의 구성을 재배치할 수 있습니다.
- **핵심 상호작용**
  - **에피소드 관리:**
    - '에피소드 리스트'에서 행의 순서를 드래그 앤 드롭으로 변경하여 **에피소드의 순서를 재구성**합니다.
    - 상단 헤더의 `+ 에피소드 추가` 버튼 또는 리스트 하단의 `+ 에피소드 추가` 버튼으로 에피소드를 추가합니다.
    - 작업 상태 변경을 통해 현재 작업 진행 상황을 알 수 있습니다.
  - **페이지 관리:**
    - '페이지 관'에서 페이지 행의 순서를 드래그 앤 드롭으로 변경하여 **콘티의 흐름을 재구성**합니다.
    - 리스트 하단의 `+ 페이지 추가` 버튼으로 페이지를 추가합니다.
  - **상태 변경:**  '상태' 드롭다운을 통해 현재 에피소드의 작업 단계를 '예정'에서 '진행중', '완료' 등으로 변경합니다.
  - **Page 화면으로 이동:** 사용자가 '페이지별 메모 리스트'에서 특정 페이지 번호나 행을 **클릭**하면, 해당 페이지의 컷을 상세하게 구성하는 **Page 관리 화면**으로 이동합니다.

## 3-4. Page 화면

- **상단 헤더:**
  - 현재 작업 중인 에피소드 타이틀과 총 페이지 수가 표시됩니다. (예: `ep.3 이웃 친구들 24p`)
  - 에피소드 메모를 확인할 수 있습니다. 페이지 작업 시 에피소드의 주요 내용을 복기할 수 있습니다.
  - 우측에는 여러 기능 버튼이 위치합니다.
    - **`+ 페이지 추가`**: 새 페이지를 생성하는 버튼입니다.
    - **`메모 패널` (가로점 3개 아이콘)**: 화면 우측에 현재 열려있는 프로젝트 관련 메모(프로젝트>에피소드>페이지>컷)를 확인할 수 있는 플로팅 패널을 열거나 닫습니다.
- **콘텐츠 1 - 페이지 리스트:**
  - **기능:** 에피소드에 포함된 모든 페이지를 작은 썸네일 카드로 보여주어 전체적인 흐름과 구성을 조망합니다.
  - **페이지 카드:** 각 카드에는 페이지의 축소 버전이 보이고, 하단 카드 인포에는 페이지 번호와 해당 페이지의 핵심 내용이 담긴 한 줄 메모가 표시됩니다. 미배정 컷메모가 있을 시 "심각도: 하" 레벨의 UI 표시가 있습니다.
  - **상호작용:**
    - 페이지 카드를 클릭하면 해당 페이지를 편집하는 **'컷 편집 보기' 모드**로 전환됩니다.
    - 페이지 내부의 컷 세부 메모를 드래그 앤 드롭으로 페이지 내부 또는 다른 페이지로 이동할 수 있습니다.
    - 페이지를 드래그 앤 드롭으로 순서를 변경할 수 있습니다. 양면 모드일 경우, 페이지 순서 변경 시 좌/우 페이지 재계산이 일어나게 됩니다.
    - `+ 페이지 추가` 카드를 통해 현재 에피소드에 새 페이지를 추가할 수 있습니다.
- **콘텐츠 2 - 페이지 상세:**
  - **기능:** 페이지에 대한 페이지 메모와 페이지 세부 메모, 페이지 내부의 컷 메모를 관리하는 영역입니다.
  - **구성:**
    - **페이지 메모:** 
      - 해당 페이지의 간단한 메모를 입력합니다. (예: "전투 씬")
      - 페이지의 감정선을 이모티콘으로 표시할 수 있습니다.
    - **페이지 세부 메모:** 해당 페이지의 세부 메모를 입력합니다. (예: A와 B가 XX에서 싸우고 있다. 건물이 부서지고 B가 던진 물건에 맞아 A의 팔이 부상을 당하고, A는 부상 당한 팔을 아파한다)
    - **컷 메모:**
      - 컷 메모 추가 버튼을 통해 컷 메모를 추가하여 컷 별 간단한 메모를 입력합니다. (예: 1 | 건물 전경, 폭발이 일어남, 2 | A와 B가 싸우고 있음, 3 | 물건을 던지는 B, 4 | A의 팔을 스침, 5 | 아파하는 A, 분해하며 B를 쳐다본다) 드래그 앤 드롭으로 순서를 변경할 수 있습니다.
      - 컷의 감정선을 이모티콘으로 표시할 수 있습니다.
- **하단 플로팅 툴바:**
  - **확대/축소 컨트롤:** 페이지의 확대, 축소, 화면 맞춤 기능을 제어합니다.
  - `양면 보기`` / ``단면 보기`: 페이지를 펼침면으로 볼지, 한 페이지씩 볼지 전환합니다.
- **핵심 상호작용**
  - 새 페이지 추가 패널 또는 새 페이지 추가 버튼을 통해 새 페이지를 추가합니다.
  - 컷 메모 추가를 통해 컷 메모를 작성할 수 있습니다.
  - **Cut 화면으로 이동:** 사용자가 특정 페이지를 **클릭**하면, 해당 페이지의 컷을 상세하게 구성하는 **Cut 관리 화면**으로 이동합니다.

## 3-5. Cut 화면

- **상단 헤더:**
  - 현재 작업 중인 에피소드 타이틀과 현재 작업 중인 페이지 번호가 표시됩니다. (예: `ep.3 이웃 친구들 1p, 2p`)
  - 페이지 메모를 확인할 수 있습니다. 컷 작업 시 페이지의 주요 내용을 복기할 수 있습니다.
  - 우측에는 여러 기능 버튼이 위치합니다.
    - 이전 페이지, 다음 페이지로 이동하는 버튼이 있습니다.
    - **`메모 패널` (가로점 3개 아이콘)**: 화면 우측에 현재 열려있는 프로젝트 관련 메모(프로젝트>에피소드>페이지>컷)를 확인할 수 있는 플로팅 패널을 열거나 닫습니다.
- **콘텐츠 1 - 중앙 캔버스:**
  - 선택된 페이지(또는 펼침면)가 실제 만화 원고지처럼 크게 표시되는 디지털 캔버스입니다.
  - **컷(Cut) 구성:** 사용자는 이 캔버스 위에서 **컷을 직접 그리고, 나누고, 합치며** 페이지의 레이아웃을 자유롭게 디자인합니다.
    - 캔버스 하단에 나타나는 **플로팅 툴바**를 사용하여 '컷 나누기', '컷 합치기', '삭제', '되돌리기' 등의 기능을 사용합니다.
  - **컷(Cut) 메모:**
    - 각 컷에는 페이지 관리 화면에서 작성한, 또는 Cut 화면에서 작성한 컷 메모를 드래그 앤 드롭으로 지정할 수 있습니다. 배정되지 않은 컷 메모와 컷에 배정된 컷 메모는 UI가 다르게 표시됩니다.(하위의 컷 세부 메모 포함)
    - 컷의 감정선을 이모티콘으로 표시할 수 있습니다.
  - **컷(Cut) 세부 메모:** 각 컷에는 `+ 컷 세부 메모 추가` 영역을 눌러 대사, 지문/나레이션, 연출 등을 입력할 수 있습니다. 아래는 작성 예시입니다.
    - **대사 입력창:** 캐릭터의 대사를 입력합니다. 말풍선 종류를 선택하는 옵션이 함께 제공될 수 있습니다. 프로젝트에서 설정한 캐릭터 정보를 연동시킬 수 있습니다.
    - **지문/나레이션 입력창:** 상황을 설명하는 지문이나 나레이션을 입력합니다.
    - **연출 메모 입력창:** 카메라 각도, 캐릭터의 표정, 효과 등 구체적인 연출 사항을 기록합니다.
    - **이미지 에셋 연결:** 컷에 사용할 배경이나 소품 등의 이미지 파일을 연결하거나 업로드하는 기능이 포함될 수 있습니다. 또는 프로젝트에서 설정한 배경 정보를 연동시킬 수 있습니다.
- **하단 플로팅 툴바:**
  - 되돌리기, 다시하기 버튼이 있습니다.
  - 컷 분할 모드 버튼이 있습니다. 클릭 시 포인터도 컷 분할모드에 맞춰 변경되고, 컷을 수직, 수평, 대각선으로 나눌 수 있습니다. 생성된 컷의 경계선에 있는 버튼을 드래그하여 크기와 형태를 조절합니다.
  - 선택 컷 삭제 버튼이 있습니다. 컷 삭제 시, 인접한 컷이 해당 컷이 차지하고 있던 크기만큼 늘어납니다.
  - 컷 초기화 버튼이 있습니다. "컷 분할 작업을 초기화 하시겠습니까?"라는 문구와, 이후 표시하지 않기 체크박스가 있는 토스트 알림이 뜹니다. 확인을 누를 시 분할된 컷이 초기화 됩니다. 컷별 세부 메모는 컷별 메모 하위로 남아있습니다.
- **핵심 상호작용**
  - **워크플로우:**
    1. 중앙 캔버스에서 컷 분할 모드를 **클릭**하여 원하는 모양으로 컷을 분할합니다.
    2. 우측 패널의 컷별 메모를 드래그 앤 드롭으로 각 컷에 지정합니다.
    3. 각 컷 내부에서, 또는 컷별 메모 하위로 입력 필드들에 **대사, 지문, 연출 내용 등의 컷 세부 메모를 입력**합니다.

# **4. 디자인 추가 설명**

**1. 전체 레이아웃 및 색상 테마**

- **기본 레이아웃:** 화면은 크게 **상단 툴바**, 중앙의 **메인 캔버스**, 그리고 메인 캔버스 내부 우측의 **상세 정보 패널**로 구성됩니다.
- **색상 테마:** 전체적으로 따뜻한 종이 질감의 베이지색(`-bg-primary: #f7f6f3`)을 배경으로 사용하며, 카드나 패널 등 주요 콘텐츠 영역은 흰색(`-bg-secondary: #fefefe`)으로 구분하여 깨끗하고 집중도 높은 환경을 제공합니다. 핵심 상호작용 요소에는 깊은 파란색(`-color-primary: #2A4D88`)이 주조색으로, 특별한 강조가 필요한 부분에는 호박색(`-color-accent: #fbc662`)이 포인트 색상으로 사용됩니다.
- **Color palette (raw values):** $deep-blue: #2A4D88; $ocean: #7C94B8; $glacial: #B1BBC8; $concerto: #D9D9D8; $amber: #fbc662; $amber-dark: #e8a83c; $amber-light: #fdd99b; $ink: #1a1a1a; $ink-soft: #4a4a4a; $ink-ghost: #9a9a9a; $paper: #f7f6f3; $white: #fefefe; $border: #e8e6e0;
- **디자인 분위기:** 
  - 80%는 미니멀/심플하게: 배경은 화이트나 아주 연한 무채색, 레이아웃은 정직한 그리드로.
  - 20%에 키치를 투척.
  - 디폴트용 썸네일은 랜덤 그라데이션.

**2. 페이지 규격**

- **페이지 썸네일:** 각 썸네일은 정적인 이미지가 아니라, **실제 페이지에 포함된 컷(Cut)들의 레이아웃 정보를 바탕으로 DOM을 단순화하여 동적으로 렌더링**됩니다. 이를 통해 사용자는 썸네일만 보고도 각 페이지의 컷 구성을 직관적으로 파악할 수 있습니다.
- **페이지-Cut화면 중앙 캔버스**
  - **디자인:** B4 규격의 실제 만화 원고지처럼 보이는 디지털 캔버스입니다. 배경은 약간 어두운 편집 보드 색상이며, 그 위에 흰색(`-bg-secondary`)의 페이지가 놓입니다.
  - **좌표 시스템 및 가이드라인:**
    - 모든 컷의 위치와 크기는 페이지의 **내곽선(Inner Frame)을 기준으로 퍼센트(%) 좌표**로 관리됩니다. 이는 페이지의 확대/축소 시에도 레이아웃 비율이 깨지지 않도록 보장합니다.
    - 레이아웃 작업을 돕기 위해 여러 가이드라인이 표시됩니다.
      - **컷이 배치되는 실제 작업 영역인 내곽선(H270mmxW180mm):** 컷 분할 전에는 파란색, 컷 분할 되어 실제 컷이 작성된 이후에는 검정색.
      - **작업의 최대 영역인 제판선(Bleed Line)(H324mmxW238mm)**

**3. 기본 설정**

- **설정의 상속 체인:** Episode>Project>User>Default 순으로 결정된다.

**4. 권한 검증 시점/비인가 접근 시점**

- **권한 검증 시점:**
  - 권한이 없을 시 리스트 미노출이기 때문에 접근이 불가능함.
  - 단, 링크로 접근했을 시, 렌더링 전 시점에서 권한을 검증하고, 편집자/댓글작성/읽기전용에 따라 UI가 변경된다.
- **비인가 접근 시점:**
  - 권한이 없는 페이지에 접근했을 시, 접근 권한이 없다는 토스트 알림을 띄우고, 확인을 누르거나 5초 후에 이전 페이지로 이동한다.

# 6. UX/UI 핵심 원칙 및 설계 지침

- **1. 사용자 여정 중심의 레이아웃 재구성 (User Flow Centric)**
  - 제시된 화면 구성은 기능적 분류일 뿐이며, 사용자가 **'아이디어 발상 → 에피소드 구성 → 페이지 콘티 → 상세 컷 작업'**으로 이어지는 워크플로우를 방해받지 않도록 화면 간의 위계와 이동 경로를 자유롭게 재설계할 수 있습니다.
  - 특정 기능이 특정 화면에 종속되기보다, 사용자가 필요로 하는 시점(Context)에 적절히 노출되도록 레이아웃을 최적화해 주세요.

- **2. 정보 설계의 위계 및 일관성 (Hierarchy & Consistency)**
  - **정보의 점진적 노출(Progressive Disclosure)**:
    - 사용자가 당장 필요하지 않은 복잡한 설정값은 숨기고(예: 상세 설정 모달, 접이식 패널),
    - 현재 작업 중인 핵심 콘텐츠(글/그림)에만 집중할 수 있는 간결한 인터페이스를 지향합니다.
  - **상속 기반의 일관성**:
    - 프로젝트-에피소드-페이지로 이어지는 계층 구조 내에서 UI 요소(버튼 위치, 아이콘 의미, 색상 규칙)의 일관성을 유지하여 학습 비용을 최소화합니다.

- **3. 직관적인 상호작용 (Intuitive Interaction)**
  - **직접 조작(Direct Manipulation):** 드래그 앤 드롭을 통한 순서 변경, 캔버스 위에서의 직접적인 컷 분할 등 사용자가 대상을 직접 건드려 제어한다는 느낌을 주어 직관성을 높입니다.
  - **시각적 피드백:** 상태 변경, 저장 완료, 미배정 메모 알림 등 시스템의 변화를 사용자가 즉각적이고 명확하게 인지할 수 있도록 시각적 장치를 배치해 주세요.

- **4. 작업 효율 극대화 (Efficiency of Use)**
  - **컨텍스트 유지:** 화면 이동 시에도 사용자가 이전에 보던 메모나 참조하던 설정이 유지되거나, '메모 패널'처럼 어디서든 호출 가능한 플로팅 요소를 활용하여 작업의 흐름이 끊기지 않게 합니다.
  - **단축 경로 제공:** 반복적인 작업(새 에피소드 추가, 컷 메모 배정 등)은 클릭 횟수를 최소화하거나 키보드 단축키, 우클릭 메뉴 등을 통해 빠르게 처리할 수 있도록 설계해 주세요.

- **5. 참고사항**
  - 제시된 기능 명세에 얽매이지 말고, 창작자가 아이디어를 원고로 실체화하는 과정에서 겪는 인지적 부하를 최소화할 수 있도록 메인 콘텐츠 영역의 구분과 기능을 사용자 편의 중심으로 재해석해 주세요.
  - **불필요한 장식은 배제하되(80%), 창작의 즐거움을 주는 키치한 포인트(20%)**가 살아있는 직관적인 툴을 지향합니다."
  - **모듈형 위젯 레이아웃:** 고정된 메인 콘텐츠 영역 대신, 사용자가 우측 사이드바나 하단 메모창의 크기를 조절하거나 고정(Pin)할 수 있는 '워크스페이스 커스텀' 기능을 도입하면 전문 작가들의 다양한 작업 스타일(글 중심 vs 그림 중심)을 모두 만족시킬 수 있습니다.
  - **리스크 감소 - 인지 과부하 방지:** 프로젝트 화면에서 너무 많은 통계와 세부 정보를 한꺼번에 보여주면 사용자가 압도될 수 있습니다. '작업 중인 에피소드로 바로가기' 같은 퀵 액션 버튼을 상단에 크게 배치하여 목표 달성 시간을 단축시키는 것이 좋습니다.
