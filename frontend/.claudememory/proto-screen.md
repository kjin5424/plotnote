>
> 각 화면의 레이아웃, 구성 요소, 상호작용, 데이터 흐름을 **구현 가능한 수준**으로 묘사한 문서입니다.

> 파일의 생성 위치는 public/prototypes/입니다. 파일은 공통 css 및 동일한 이름의 html+css로 제작합니다.

- **UX/UI 핵심 원칙 및 설계 지침**
  - **1. 사용자 여정 중심의 레이아웃 재구성 (User Flow Centric)**
    - 제시된 화면 구성은 기능적 분류일 뿐이며, 사용자가 **'아이디어 발상 → 에피소드 구성 → 페이지 콘티 → 상세 컷 작업'**으로 이어지는 워크플로우를 방해받지 않도록 화면 간의 위계와 이동 경로를 자유롭게 재설계할 수 있습니다.
    - 특정 기능이 특정 화면에 종속되기보다, 사용자가 필요로 하는 시점(Context)에 적절히 노출되도록 레이아웃을 최적화해 주세요.
  - **2. 정보 설계의 위계 및 일관성 (Hierarchy & Consistency)**
    - **정보의 점진적 노출(Progressive Disclosure)**:
      - 사용자가 당장 필요하지 않은 복잡한 설정값은 숨기고(예: 상세 설정 모달, 접이식 패널),
      - 현재 작업 중인 핵심 콘텐츠(글/그림)에만 집중할 수 있는 간결한 인터페이스를 지향합니다.
    - **상속 기반의 일관성**:
      - 프로젝트-에피소드-페이지로 이어지는 계층 구조 내에서 UI 요소(버튼 위치, 아이콘 의미, 색상 규칙)의 일관성을 유지하여 학습 비용을 최소화합니다.
  - **3. 직관적인 상호작용 (Intuitive Interaction)**
    - **직접 조작(Direct Manipulation):** 드래그 앤 드롭을 통한 순서 변경, 캔버스 위에서의 직접적인 컷 분할 등 사용자가 대상을 직접 건드려 제어한다는 느낌을 주어 직관성을 높입니다.
    - **시각적 피드백:** 상태 변경, 저장 완료, 미배정 메모 알림 등 시스템의 변화를 사용자가 즉각적이고 명확하게 인지할 수 있도록 시각적 장치를 배치해 주세요.
  - **4. 작업 효율 극대화 (Efficiency of Use)**
    - **컨텍스트 유지:** 화면 이동 시에도 사용자가 이전에 보던 메모나 참조하던 설정이 유지되거나, '메모 패널'처럼 어디서든 호출 가능한 플로팅 요소를 활용하여 작업의 흐름이 끊기지 않게 합니다.
    - **단축 경로 제공:** 반복적인 작업(새 에피소드 추가, 컷 메모 배정 등)은 클릭 횟수를 최소화하거나 키보드 단축키, 우클릭 메뉴 등을 통해 빠르게 처리할 수 있도록 설계해 주세요.
  - **5. 참고사항**
    - 제시된 기능 명세에 얽매이지 말고, 창작자가 아이디어를 원고로 실체화하는 과정에서 겪는 인지적 부하를 최소화할 수 있도록 메인 콘텐츠 영역의 구분과 기능을 사용자 편의 중심으로 재해석해 주세요.
    - **모듈형 위젯 레이아웃:** 고정된 메인 콘텐츠 영역 대신, 사용자가 우측 사이드바나 하단 메모창의 크기를 조절하거나 고정(Pin)할 수 있는 '워크스페이스 커스텀' 기능을 도입하면 전문 작가들의 다양한 작업 스타일(글 중심 vs 그림 중심)을 모두 만족시킬 수 있습니다.
    - **리스크 감소 - 인지 과부하 방지:** 프로젝트 화면에서 너무 많은 통계와 세부 정보를 한꺼번에 보여주면 사용자가 압도될 수 있습니다. '작업 중인 에피소드로 바로가기' 같은 퀵 액션 버튼을 상단에 크게 배치하여 목표 달성 시간을 단축시키는 것이 좋습니다.

## 1. 공통 셸 (App Shell)

전체 화면은 **사이드바(좌측) + 메인 콘텐츠(우측)**의 2단 구조다.

### 1-1. 사이드바 — 펼친 상태 (너비 약 240px)

사이드바는 `$white` 배경에 우측으로 `$border` 1px 구분선이 있다.

**최상단 영역:**
로고가 좌측에 위치한다. PlotNote의 심볼 마크(작은 펜촉+노트 형상)와 "PlotNote" 워드마크가 나란히 놓인다. 로고 우측 끝에 사이드바 접기/펼치기 토글 버튼(`keyboard_double_arrow_left_20dp_000000_FILL0_wght400_GRAD0_opsz20.svg`)이 있다. 이 버튼을 누르면 사이드바가 44px 레일로 축소된다.

**검색 영역:**
로고 바로 아래에 검색 입력 필드가 있다. 연한 `$paper` 배경에 돋보기 아이콘과 "검색..." 플레이스홀더 텍스트가 보인다. 이 필드를 클릭하면 화면 정중앙에 **검색 모달**이 오버레이로 뜬다.

**트리 메뉴 영역:**
검색 아래에 세로로 긴 트리 메뉴가 펼쳐진다. 현재 사용자가 위치한 경로의 트리 노드는 `$deep-blue` 텍스트 + 좌측에 2px 두께의 `$deep-blue` 세로 바가 들어가 하이라이트된다. 좌측 삼각형▼/▶(`keyboard_arrow_right_20dp_000000_FILL0_wght400_GRAD0_opsz20.svg`) 토글로 접거나 펼칠 수 있다.

트리 메뉴는 세 개의 그룹으로 나뉜다.

- 첫 번째 그룹은 **"즐겨찾기"**다. 즐겨찾기(`bookmark_20dp_000000_FILL0_wght400_GRAD0_opsz20.svg`) 아이콘과 "즐겨찾기" 레이블이 있다. 펼치면 즐겨찾기 지정한 에셋(프로젝트/에피소드/페이지)들이 들여쓰기 되어 나열된다.
- 두 번째 그룹은 **"나의 책장"**이다. 홈 아이콘(`house_20dp_000000_FILL0_wght400_GRAD0_opsz20.svg`)과 "나의 책장" 레이블이 있다. 펼치면 내가 생성한 모든 프로젝트가 나열된다. 각 프로젝트 행을 클릭하면 해당 프로젝트 하위의 에피소드 목록이 들여쓰기 되어 추가로 펼쳐지고, 에피소드를 클릭하면 페이지 목록이 다시 들여쓰 기되어 나타난다. 페이지까지가 트리의 최하위 노드이며, 컷은 트리에 표시되지 않는다.
- 세 번째 그룹은 **"공유 책장"**이다. 공유 아이콘(`diversity_3_20dp_000000_FILL0_wght400_GRAD0_opsz20.svg`)과 "공유 책장" 레이블이 있다. 다른 사용자가 공유해 준 프로젝트들이 동일한 트리 구조로 나열된다.

**최하단 영역:**
사이드바 바닥에 회원 정보가 표시된다. 원형 아바타(기본은 이니셜 레터 배지, `$ocean` 배경 + 흰색 텍스트)와 닉네임(예: "베베")이 있다.
클릭시 사용자 설정을 할 수 있는 모달이 표시된다. (Phase 3 로그인 로그아웃 기능.)

- **기본 페이지 수:** 숫자 입력(기본값 24). 좌우에 -/+ 스테퍼 버튼.
- **읽기 방향:** "좌→우 (LTR)" / "우→좌 (RTL)". RTL이 기본
- **기본 뷰:** "단면" / "양면". 양면이 기본 선택.
- **시작 페이지 (양면 선택 시에만 활성화):** "홀수 페이지 시작" / "짝수 페이지 시작". 홀수가 기본.

### 1-2. 사이드바 — 접힌 상태 (너비 44px, "레일" 모드)

사이드바가 접히면 44px 너비의 세로 레일이 된다. 텍스트는 사라지고 아이콘만 세로로 나열된다.

최상단에 PlotNote 심볼 마크(펜촉 아이콘만). 그 아래로 Current로 설정된 에셋까지의 아이콘이 세로로 정렬된다. 내비게이션 아이콘 표시 단계는 화면이 깊게 들어갈수록 하단에 아이콘이 하나씩 추가되는 방식이다.(예: 컷관리- 심볼→홈→에피소드→페이지→컷) 현재 사용자가 위치한 계층의 아이콘은 `$deep-blue`로 채워지고, 나머지는 `$ink-ghost`다.
아이콘이 유지되거나 사라지는 기준은 **'상위 데이터의 변경 여부'**이다. 하위 단계에서 다시 상위 단계로 이동하더라도, 이미 'Current'로 설정된 정보가 바뀌지 않았다면 아이콘은 그대로 유지된다. 상위 에셋(예: 책, 에피소드 등)을 다른 것으로 변경하면, 그 아래에 종속되었던 모든 하위 에셋의 'Current' 상태가 깨지기 때문에 아이콘이 초기화 된다.(예: 'A 에피소드'의 컷을 보다가 'B 에피소드'로 이동하는 순간, 아이콘은 에피소드까지만 표시된다.)

**핵심 인터랙션:** 레일 모드의 아이콘 위에 마우스를 올리면, 해당 아이콘 우측으로 **팝업 패널**이 나타난다. 예를 들어 "나의 책장" 아이콘에 호버하면, 우측으로 200px 폭의 패널이 슬라이드 아웃되며 소속 프로젝트 리스트가 보인다. 프로젝트에 호버하면 에피소드 리스트가, 에피소드에 호버하면 페이지 리스트가 나타난다. 특정 행을 클릭하면 해당 화면으로 이동한다. 마우스가 팝업 영역을 벗어나면 패널이 사라지나, 이 사라지기까지의 속도를 조절해서 마우스 터널링 현상을 해결한다. 단, 마우스가 다른 아이콘에 호버하면 바로 기존 팝업 패널을 지우고 새로운 팝업 패널을 띄운다.

최하단에 원형 아바타 아이콘만 표시된다.

### 1-3. 메인 콘텐츠 영역

사이드바 우측의 나머지 전체 영역이 메인 콘텐츠다. `$paper` 배경. 화면별로 **상단 헤더**, **콘텐츠 본문**, 그리고 필요 시 **하단 플로팅 툴바**나 **우측 사이드 패널**이 조합된다. 각 화면의 상세 구성은 아래에서 기술한다.

## 2. Bookshelf 화면 (책장)

사이드바 트리에서 "나의 책장" 또는 "공유 책장" 루트를 선택하거나, 최상위 경로(/)에 진입하면 이 화면이 표시된다.

### 2-1. 상단 헤더

높이 56px 정도의 가로 바. `$white` 배경, 하단에 `$border` 1px 구분선.

좌측에 **"plotnote"**이라는 화면 타이틀이 표시된다.

우측에 **"+ 새 프로젝트"** 버튼이 있다. `$deep-blue` 배경에 `$white` 텍스트, 좌측에 + 아이콘. border-radius 8px의 필(filled) 버튼이다. 호버 시 약간 밝아진다.

### 2-2. 콘텐츠 본문

본문은 세로로 스크롤되는 영역이다. `$paper` 배경 위에 세 개의 섹션이 순서대로 쌓인다. 섹션 헤더는 하위 에셋이 있을 시 기본 펼쳐진 상태이다. 섹션은 호버 시 배경이 살짝 밝아진다.

- **섹션 1: 즐겨찾기**
  섹션 헤더 행이 있다. 좌측에 북마크 아이콘 `$amber` 색상, "즐겨찾기" 레이블 `$ink` 14px 세미볼드, 그리고 프로젝트 개수가 `$ink-ghost` 색상의 작은 원형 뱃지(예: "3")로 표시된다. 행 우측 끝에 ▼/▶ 접기·펼치기 토글이 있다.
  섹션 헤더를 클릭하면 아래에 프로젝트 카드들이 **그리드**로 나타난다. 한 행에 3~4개의 카드가 나열되고(반응형으로 화면 폭에 따라 조정), 카드 간 간격은 16px이다.
- **섹션 2: 나의 책장**
  즐겨찾기와 동일한 구조. 책 아이콘 + "나의 책장" + 카운트 뱃지. 내가 소유한 모든 프로젝트가 카드 그리드로 나열된다. 이 섹션의 **맨 마지막 위치**에 "**`+ 새 프로젝트`**" 카드가 있다. 이 카드는 점선 보더(`$concerto` 색상, 2px dashed), 중앙에 큰 + 아이콘과 "새 프로젝트" 텍스트가 세로로 놓인 빈 카드다. 호버 시 점선이 `$ocean`으로 변하고 배경이 살짝 밝아진다.
- **섹션 3: 공유 책장**
  공유 아이콘 + "공유 책장" + 카운트 뱃지. 다른 사용자가 공유해 준 프로젝트만 표시된다. "`+ 새 프로젝트`" 카드는 없다.

### 2-3. 프로젝트 카드 상세 구조

각 카드는 `$white` 배경, `$border` 1px 보더, border-radius 12px, shadow-sm의 카드 컨테이너다. 카드 폭은 그리드 컬럼에 맞춰 유동적이고, 세로 비율은 대략 4:5 정도다.

- **카드 상단 — 썸네일 영역 (전체 높이의 약 60%):**
  프로젝트에 대표 이미지가 있으면 이미지가 `object-fit: cover`로 채워진다. 이미지가 없으면 **랜덤 그라디언트 배경**이 표시된다. 그라디언트는 프로젝트 ID를 시드로 하여 2~3가지 파스텔 톤 색상이 대각선 또는 원형으로 조합된다 — 이것이 "키치 20%"를 담당한다.
- **카드 하단 — 정보 영역 (전체 높이의 약 40%):**
  padding 12px.
  첫 줄에 프로젝트 제목이 `$ink` 15px 세미볼드로 표시된다. 한 줄을 넘기면 말줄임(...) 처리. 제목 행의 우측 끝에 **즐겨찾기 토글 버튼**이 있다. 비활성 시 빈 북마크 `$concerto` 색상, 활성 시 채워진 북마크 `$amber` 색상. 클릭 시 토글되며, 활성화하면 "즐겨찾기" 섹션에도 이 카드가 나타난다.
  두 번째 줄에 메타데이터가 `$ink-ghost` 12px로 표시된다. "마지막 수정: 3일 전" 형식의 상대 시간. 7일 초과 시 절대 시간(locale에 따라 다른 YYYY-MM-DD HH24:MI”)으로 표시. 총 페이지 수.
  세 번째 줄에 참여자 뱃지가 가로로 나열된다. 각 뱃지는 작은 원형 아바타(20px)가 겹쳐지는 형태이고, 참여자가 3명을 넘으면 "+2" 형태의 오버플로우 뱃지가 표시된다.

### 2-4. 새 프로젝트 생성 모달

"`+ 새 프로젝트`" 버튼 또는 "`+ 새 프로젝트`" 카드 클릭 시 화면 중앙에 모달이 뜬다.

모달은 `$white` 배경, border-radius 16px, shadow-xl. 폭 480px 정도. `$paper` 반투명 오버레이가 뒤를 덮는다.

모달 헤더에 "새 프로젝트 만들기" 타이틀과 우측에 X 닫기 버튼.

모달 본문에 다음 입력 필드가 세로로 배치된다. 입력 필드의 가로 크기는 동일하다.

- **프로젝트 이름:** 텍스트 입력 필드. 플레이스홀더 "프로젝트 이름을 입력하세요".
- **기본 페이지 수:** 숫자 입력(기본값 24). 좌우에 -/+ 스테퍼 버튼.
- **읽기 방향:** "좌→우 (LTR)" / "우→좌 (RTL)". RTL이 기본 선택.
- **기본 뷰:** "단면" / "양면". 양면이 기본 선택.
- **시작 페이지 (양면 선택 시에만 활성화):** "홀수 페이지 시작" / "짝수 페이지 시작". 홀수가 기본.

모달 하단에 **"만들기"** 버튼(`$deep-blue` 필 버튼)과 **"취소"** 버튼(텍스트 버튼)이 우측 정렬로 나란히 놓인다.

### 2-5. 핵심 상호작용

프로젝트 카드를 클릭하면 해당 **Project 화면**으로 라우팅된다. URL은 `/project/:projectId` 형태. 사이드바 트리에서도 해당 프로젝트 노드가 자동으로 하이라이트된다.

## 3. Project 화면 (프로젝트)

사이드바 트리에서 특정 프로젝트를 선택하거나, 책장 화면에서 프로젝트 카드를 클릭하면 이 화면으로 진입한다.

### 3-1. 상단 헤더

높이 56px. `$white` 배경, 하단 `$border` 1px.

**좌측:** 프로젝트 제목이 20px 세미볼드 `$ink`로 표시된다(예: "고양이와 강아지"). 제목은 인라인 편집 가능 — 클릭하면 텍스트 커서가 나타나 바로 수정할 수 있다. 제목 바로 우측에 **즐겨찾기 토글**이 있다.

**우측:** 세 개의 버튼이 가로로 나열된다.

1. 콘텐츠 본문을 클릭에 따라 상단하단 구성 / 좌측우측구성으로 변경하여 볼 수 있는 버튼.
2. **"`+ 에피소드 추가`"** — `$deep-blue` 필 버튼.
3. **"`메모 패널`"** — ⋯(가로 점 3개) 아이콘의 고스트 버튼. 클릭 시 우측에서 메모 플로팅 패널이 슬라이드 인/아웃된다.

**하단:** 프로젝트 제목 아래에 **프로젝트 메모 미리보기**가 `$ink-ghost` 12px 한 줄(말줄임 처리)로 표시된다. 상단과는 border로 구분된다. 빈 상태에서는 "이 프로젝트의 전체 줄거리를 기록하세요..." 플레이스홀더가 `$ink-ghost`로 표시된다. 포커스되면 좌측에 2px `$ocean` 세로 바가 나타난다. 프로젝트의 전체 줄거리나 핵심 내용을 자유롭게 기록한다. 이 미리보기를 클릭하면 프로젝트 메모 전문이 작은 팝오버로 뜬다.

### 3-2. 콘텐츠 본문 — 상단 프로젝트 정보 영역

본문은 세로 스크롤 영역이다. 프로젝트 설정은 상단 절반 정도를 차지하는 "프로젝트 정보" 블록과, 하단의 "에피소드 목록" 블록으로 나뉜다. 두 블록 사이에 `$border` 수평 구분선이 있다.

**프로젝트 정보 블록**은 내부적으로 다시 2개의 카드/섹션으로 구성된다. 이 카드들은 2열 그리드로 배치된다.

- **카드 A: 프로젝트 정보/통계 (상단)**`$white` 배경 카드. "프로젝트 정보" 라벨. 내부에 가로로 정보 행이 나열된다.
  - 총 에피소드: 12화
  - 총 페이지: 288p
  - 마지막 작업: 3일 전
    각 행은 좌측에 라벨 `$ink-ghost`, 우측에 값 `$ink`로 정렬된다.
  - **"공유"** 버튼이 있다. `$ocean` 아웃라인 버튼. 클릭 시 공유 모달이 뜨며, 현재 공유 중인 멤버 리스트(이름, 역할 뱃지 — 편집자/댓글/읽기전용)가 보이고, 링크 공유 입력필드와 복사 버튼이 있다. (Phase 3: text 또는 pdf 내보내기 버튼)
  - **"설정"** 버튼이 있다. 클릭 시 카드 A가 아래로 확장되며 카드 B의 내용이 카드 A 아래로 표시된다.
- **카드 B: 프로젝트 설정 (카드 A 내부 하단)**`$white` 배경 카드. "프로젝트 설정" 라벨. 내부에 설정 행이 나열된다.
  - 기본 페이지 수: 24 (드롭다운 또는 스테퍼)
  - 읽기 방향: RTL / LTR (토글 스위치)
  - 기본 뷰: 단면 / 양면 (토글 스위치)
  - 시작 페이지: 홀수 / 짝수 (양면 선택 시에만 활성화)
    설정을 변경하면 즉시 저장되고, 해당 프로젝트 하위의 에피소드 중 개별 설정이 없는 에피소드들에 자동 반영된다(설정 상속 체인: Episode > Project > User > Default).
- **카드 C: 프로젝트 세부 (하단)**`$white` 배경 카드. 상단에 "프로젝트 설정" 라벨. 상단 우측에는 마인드맵 아이콘이 있고, 클릭 시 모달이 표시되며, 모달에서는 각 프로젝트 세부 메모에 대한 메모를 작성할 수 있는 메모툴이 뜬다. 내부는 **태그 기반 리스트**다.
  - 각 행은 "#태그명 | 내용" 형식으로 표시된다. 예: "#장르 | SF", "#캐릭터 | 주인공, 이름 홍길동, 나이 18", "#복선 | 아버지가 사실 진짜 친아버지이다".
  - 각 행의 태그명 부분은 `$deep-blue` 배경 `$white` 텍스트의 작은 라운드 뱃지, 내용 부분은 일반 텍스트다. 태그명 부분 클릭 시 작은 전용 팝업이 표시되고, 위에는 입력창, “태그를 입력해주세요” 플레이스 홀더가 있으며, 하단에는 디폴트 태그들과 사용자가 프로젝트 세부 메모 내에서 설정한 이력이 있는 태그들이 나타난다. 하단의 태그들은 우측에 삭제 아이콘이 있어 클릭 시 태그 추천 목록에서 삭제된다.
  - 행 우측에 삭제 아이콘 버튼이 호버 시에만 나타난다.
  - 리스트 맨 아래에 "`+ 세부 정보 추가`" 텍스트 버튼이 있다. 텍스트 버튼 클릭 시 태그에 focus가 가게 된다.
- **카드 D: 프로젝트 세부 마인드맵 (모달):** `$white` 배경 모달. 프로젝트 세부설정은 Kinopio와 같은 형태의 메모 툴이다. 프로젝트 세부에 작성한 내용을 바탕으로 Kinopio([https://kinopio.club/hello-kinopio-g8_KXnyGRYwqwZ7ZauFLs](https://kinopio.club/hello-kinopio-g8_KXnyGRYwqwZ7ZauFLs)) 또는 마인드맵 형식으로 세부 내용에 대한 세부 메모를 작성할 수 있다.
  (Phase 1: 구현x, Phase 2: 기본 노드 드래그 배치, Phase 3: 연결선+자동 레이아웃)

### 3-3. 콘텐츠 본문 — 하단 에피소드 목록 영역

"에피소드 목록" 섹션 헤더가 있다. 좌측에 "에피소드 목록"(16px 세미볼드), 에피소드 개수 뱃지, 우측에 **정렬 버튼 아이콘 버튼**(옵션: "오름차순", “내림차”)이 있다. 클릭 시 옵션이 토글된다.

그 아래로 에피소드들이 **행(row) 기반 리스트**로 나열된다. 각 행은 `$white` 배경에 `$border` 하단 보더, padding 12px 16px, 높이 약 64px이다.

**에피소드 행 구성 (좌→우):**

1. **드래그 핸들:** 드래그 핸들 아이콘에 `$concerto` 색상. 마우스 커서가 grab 모양으로 변하며, 드래그 시 행이 반투명해지면서 드래그 고스트가 따라온다. 드롭하면 순서가 변경된다.
2. **에피소드 번호와 제목:** "1화"가 `$ocean` 색상 12px 모노스페이스, 바로 옆에 제목(예: "첫 만남")이 `$ink` 15px 세미볼드. 제목은 인라인 편집 가능.
3. **시놉시스:** 제목 우측 한 줄(기본, 내용이 길어질 시 행이 늘어남)로 표시되는 에피소드 요약 메모. `$ink-soft` 13px. 인라인 편집 가능하며, 빈 상태에서는 "줄거리를 입력하세요..." 플레이스홀더가 `$ink-ghost`로 표시.
4. **상태 뱃지:** 행 우측 영역. 라운드 뱃지로 표시.
   - "예정" — `$glacial` 배경 `$ink-soft` 텍스트
   - "진행중" — `$ocean` 배경 `$white` 텍스트
   - "완료" — `$deep-blue` 배경 `$white` 텍스트 + ✓ 체크 아이콘
5. **메타데이터:** "24p · 3일 전"이 `$ink-ghost` 12px로 표시.
6. **더보기 메뉴:** ⋮(세로 점 3개) 아이콘 고스트 버튼. 클릭 시 드롭다운 메뉴가 나타나며 "에피소드 설정 변경", "삭제" 등의 옵션이 있다.

**리스트 최하단:** "**`+ 에피소드 추가`**" 행이 있다. 점선 보더에 + 아이콘과 텍스트. 클릭 시 리스트 맨 아래에 새 행이 추가되고, 제목 필드에 자동 포커스된다.

### 3-5. 메모 플로팅 패널 (우측)

헤더의 ⋯ 버튼으로 토글되는 **우측 플로팅 패널**. 메인 콘텐츠 영역의 우측에서 슬라이드 인되며, 폭 320px, `$white` 배경, 좌측에 `$border` 1px 보더, shadow-lg.

패널 내부는 현재 프로젝트의 계층 메모를 트리 형태로 보여준다. 프로젝트 메모 → 에피소드별 메모 → 페이지별 메모 → 컷별 메모가 들여쓰기와 접기/펼치기(디폴트: 접기)로 탐색 가능하다. 읽기 전용 모드이며, 각 메모 행을 클릭하면 해당 화면으로 이동한다.

### 3-6. 핵심 상호작용

에피소드 행을 클릭하면 해당 **Episode 화면**으로 이동한다. URL은 `/project/:projectId/episode`로, 선택한 에피소드가 우측 상세 패널에 자동 표시된다.

## 4. Episode 화면 (에피소드)

프로젝트 화면에서 에피소드를 선택하거나, 사이드바에서 에피소드 노드를 클릭하면 이 화면으로 진입한다.

### 4-1. 상단 헤더

높이 56px. `$white` 배경.

**좌측:** 프로젝트 제목(예: "고양이와 강아지")이 `$ink` 18px로 표시되고, 그 옆에 `$concerto` 색상의 "|" 구분자, 그 뒤에 "에피소드 리스트"가 `$ink-ghost` 14px로 표시된다. 이를 통해 Project 화면과의 차이를 명확히 한다. 프로젝트 제목 아래에 **프로젝트 메모 미리보기**가 `$ink-ghost` 12px 한 줄(말줄임 처리)로 표시되어, 에피소드 작업 중 프로젝트의 주요 내용을 복기할 수 있다. 이 미리보기를 클릭하면 프로젝트 메모 전문이 작은 팝오버로 뜬다.

**우측:**

1. **"`+ 에피소드 추가`"** — `$deep-blue` 필 버튼.
2. **"`메모 패널`"** — ⋯ 아이콘 고스트 버튼.

### 4-2. 콘텐츠 본문 — 좌측: 에피소드 네비게이터 (전체 폭의 약 35%)

본문은 **좌우 2분할**이다. 좌측은 에피소드 목록 네비게이터, 우측은 선택된 에피소드의 상세 내용이다. 좌우 경계에 리사이즈 핸들이 있어 비율 조정이 가능하다. 좌측 에피소드 네비게이터는 아예 접을 수도 있다(약 44px). 접을 시 맨 위에는 네비게이터 펼치기 버튼, 아래로는 1, 2, 3, 4, …와 같은 에피소드 번호가 동그란 네모 형태로 표시되고 총 페이지 수가 많을 경우 해당 영역에서 세로로 쭉 나열되면서 스크롤이 가능하며, 클릭 시 해당 에피소드의 에피소드 상세를 수정할 수 있다.

**상단 탭 바:** 네비게이터 최상단에 4개의 필터 탭이 가로로 나열된다 — "전체", "진행중", "완료", "예정". 현재 선택된 탭은 `$deep-blue` 텍스트 + 하단 2px `$deep-blue` 언더바. 비선택 탭은 `$ink-ghost`.

탭 우측에 **정렬 버튼**(아이콘)이 있어 오름차순 / 내림차순으로 전환할 수 있다.

**에피소드 리스트:** 탭 아래로 에피소드 행이 세로로 나열된다. 각 행은 Project 화면의 에피소드 행과 유사하지만, 추가로 **작은 썸네일** (40x54px 정도, B4 비율)이 행 맨 왼쪽에 표시된다. 썸네일은 에피소드의 맨 첫번째 페이지을 표시하고, 맨 첫번째 페이지에 컷작업이 되어있지 않은 경우 밝은 컬러의 랜덤 그라디언트로 생성된다. 현재 선택된(우측 상세에서 편집 중인) 에피소드 행은 `$paper` 배경 + 좌측 3px `$deep-blue` 세로 바로 하이라이트된다.

행 구성: 드래그 핸들 | 썸네일 | (에피소드 번호 + 제목 + 한 줄 메모) | 페이지 수 | 상태 뱃지.

**최하단:** "`+ 에피소드 추가`" 행.

### 4-3. 콘텐츠 본문 — 우측: 에피소드 상세 (전체 폭의 약 65%)

우측 패널의 상단에 선택된 에피소드의 **서브 헤더**가 있다.

**서브 헤더:** 좌측에 "3화 이웃 친구들" (번호 `$ocean`, 제목 `$ink` 18px 세미볼드), 그 옆에 "24p" 페이지 수 뱃지. 우측에 **상태 드롭다운** (현재 상태 뱃지를 클릭하면 "예정/진행중/완료" 선택 가능), **개별 설정 버튼** (톱니바퀴 아이콘), **공유 버튼**이 있다. (Phase 3: text 또는 pdf 내보내기 버튼)

상태를 "완료"로 변경할 때 미배정 컷 메모가 있으면 **토스트 알림**이 화면 하단에서 슬라이드 업된다. "미배정 컷 메모가 N개 있습니다. 그래도 완료 처리하시겠습니까?" + "취소" / "미배정 메모 삭제 후 완료" / "그대로 완료" 세 버튼.

### 4-4. 에피소드 상세

**블록 A: 에피소드 메모**
"에피소드 메모" 라벨. contenteditable 텍스트 영역. 에피소드의 전반적 시놉시스를 기록한다. 포커스 시 `$ocean` 좌측 세로 바.

**블록 B: 에피소드 세부 메모**
"세부 메모" 라벨. 프로젝트 세부와 동일한 "내용 #태그" 형식의 추가형 리스트. 다른 점은 프로젝트는 카테고리 분류 목적에 가까웠으나(단, 카테고리 미지정 가능) 에피소드는 해시태그를 추가하여 관리하기를 권장하는 형식이다. **"#"** 입력 시 작은 전용 팝업이 표시되어, 과거 에피소드 세부 메모 내에서 기입한 적이 있는 태그가 추천으로 노출되며, 이는 화살표 버튼으로 이동하여 엔터로 선택할 수 있고, 텍스트 입력 시 실시간 부분 일치 검색어가 필터링 된다. 태그들은 우측에 삭제 아이콘이 있어 클릭 시 태그 추천 목록에서 삭제된다. 예: "과거 회상씬에서 아버지로 유추되는 사람이 보임 #복선".

**블록 C: 페이지별 메모 리스트**
"페이지별 메모" 라벨과 총 페이지 수(예: "24 페이지"). 우측에는 탭 보기 아이콘이 있으며, 해당 아이콘 클릭 시 A와 B 우측으로 C가 위치하게 된다. 다시 클릭할 시 좌우배치(좌: A+B, 우: C)에서 세로 배치(A→B→C)로 되돌릴 수 있다. 아래에 페이지 행이 세로로 나열된다. 각 행:

- **드래그 핸들**
- **페이지 번호** "p.01" ~ "p.24" — `$ocean` 색상 12px 모노스페이스.
- **한 줄 메모 입력 필드** — `$paper` 배경의 인라인 텍스트 필드. "페이지 메모를 입력하세요..." 플레이스홀더. 이 필드에 입력한 내용은 Page 화면의 해당 페이지 메모와 **양방향 연동**된다.
- **감정 이모티콘 선택기** — 작은 원형 버튼을 클릭하면 이모지 피커가 뜨고, 선택한 이모지가 메모 앞에 표시된다. (스키마는 현제 화면별 상세 묘사 즉, 프로토타입을 기준으로 수정될 예정이다.)

행은 드래그 앤 드롭으로 순서 변경 가능하다. 순서가 바뀌면 실제 페이지 순서도 변경되며, 양면 뷰의 좌/우 페이지도 자동 재계산된다.

리스트 최하단에 "`+ 페이지 추가`" 행이 있다.

### 4-5. 핵심 상호작용

페이지별 메모 리스트에서 특정 페이지 행의 번호("p.05")를 클릭하면 해당 **Page 화면**으로 이동한다. URL: `/project/:projectId/episode/:episodeId/page`.

## 5. Page 화면 (페이지)

에피소드 화면에서 페이지를 선택하거나, 사이드바 트리에서 페이지 노드를 클릭하면 이 화면으로 진입한다.

### 5-1. 상단 헤더

높이 56px. `$white` 배경.

**좌측:** 에피소드 타이틀 + 총 페이지 수가 표시된다. 예: "3화 이웃 친구들 ( 24p )". "3화" 부분은 `$ocean` 14px, "이웃 친구들"은 `$ink` 18px 세미볼드, "( 24p )"는 `$ink-ghost` 14px. 아래에 **에피소드 메모 미리보기**가 `$ink-ghost` 12px 한 줄로 표시되어, 페이지 작업 중 에피소드의 주요 내용을 복기할 수 있다. 이 미리보기를 클릭하면 에피소드 메모 전문이 작은 팝오버로 뜬다.

**우측:**

1. **"`+ 페이지 추가`"** — `$deep-blue` 필 버튼.
2. **"`메모 패널`"** — ⋯ 아이콘 고스트 버튼.

### 5-2. 콘텐츠 본문 — 좌측: 페이지 썸네일 그리드 (전체 폭의 약 20~30%)

본문은 다시 **좌우 분할**이다. 좌측이 페이지 카드 그리드, 우측이 선택된 페이지의 상세 정보 패널이다.

**페이지 카드 그리드:**`$paper` 배경 위에 페이지 카드들이 그리드로 배치된다. 뷰 모드에 따라 레이아웃이 달라진다. 좌우 경계에 리사이즈 핸들이 있어 비율 조정이 가능하다. 좌측 페이지 썸네일 그리드는 아예 접을 수도 있다. 접을 시 맨 위에는 그리드 펼치기 버튼, 아래로는 양면/단면 뷰 관계 없이 1, 2, 3, 4, …와 같은 페이지 번호가 동그란 네모 형태로 표시되고 총 페이지 수가 많을 경우 해당 영역에서 스크롤이 가능하며, 클릭 시 해당 페이지의 페이지 상세를 수정할 수 있다. 양면뷰는 스프레드 한장, 단면 뷰는 페이지 한장이 표시된다.(이렇게 해도 20~30%는 좁은가?)

- **양면(spread) 뷰:** 두 장의 페이지 카드가 나란히 붙어 하나의 "스프레드"를 이룬다. 스프레드 간 간격은 24px, 스프레드 내 두 카드 사이는 0~2px(책이 펼쳐진 것처럼). 한 행에 2~3개의 스프레드가 나열된다. 홀수 시작일 경우 첫 번째 스프레드의 좌측은 **빈 커버 카드**(회색 빗금 또는 "표지" 텍스트가 들어간 비활성 카드)가 놓이고 우측에 p.01이 온다. 양면뷰여도 페이지 상세는 한 페이지씩 수정이 가능하다.
- **단면(single) 뷰:** 한 장씩 독립적으로 나열된다. 한 행에 5~6개의 카드가 배치된다.

**Ctrl+스크롤:** 마우스 휠을 Ctrl과 함께 굴리면 행당 카드 수가 변하며 확대/축소된다.

**개별 페이지 카드 구조:**

카드는 `$white` 배경, `$border` 1px 보더, 세로 비율 B4(238:324).

- **카드 상단 — 페이지 썸네일 영역:** 전체 높이의 약 80%. 이 영역에는 **정적 이미지가 아니라, 해당 페이지에 포함된 컷들의 레이아웃 정보를 바탕으로 DOM을 단순화하여 동적으로 렌더링**한다. 즉, 컷이 4개로 나뉘어 있으면 4개의 영역이 보더로 구분되어 보이고, 각 컷 안에 컷 메모의 첫 줄이 아주 작은 텍스트로 표시된다(너무 길면 말줄임). 컷이 미분할 상태(기본 전체 1컷)이면 빈 페이지처럼 보인다.
  미배정 컷 메모가 있는 페이지에는 카드 우측 상단에 작은 **경고 도트** (`$amber` 6px 원)가 표시된다 — "심각도: 하" 수준의 시각적 힌트.
- **카드 하단 — 정보 영역:** 전체 높이의 약 20%. 좌측에 페이지 번호(예: "p.05") `$ocean` 12px, 우측에 페이지 메모 미리보기가 `$ink-soft` 11px 한 줄 말줄임으로 표시된다.

**상호작용:**

- 카드 **호버** 시 컷 영역 정 중앙에 작은 “컷 편집” 버튼이 있다. 해당 버튼을 누를 시 Cut 화면으로 이동할 수 있다.
- 카드를 **클릭**하면 우측:페이지 상세 패널에서 해당 페이지로 스크롤이 이동한다.
- 카드를 **드래그 앤 드롭**으로 순서를 변경할 수 있다.

**그리드 최후단:** "**`+ 페이지 추가`**" 카드. 점선 보더 + 큰 + 아이콘.

### 5-3. 페이지 썸네일 그리드 영역 하단 툴바

좌측:썸네일 그리드 영역의 하단의 가로 바. `$white` 배경, shadow-lg, border-radius 12px, padding 4px 12px.

좌측에:

- **확대**(🔍+) / **화면 맞춤**(⛶) / **축소**(🔍-) 아이콘 버튼.
- 현재 줌 레벨 텍스트 (예: "100%").
- 기본 양면 또는 단면이 화면에 꽉 차는 크기가 화면 맞춤의 기본. 확대 축소를 통해 보이는 페이지의 개수가 달라진다.

우측에:

- **"양면 보기" / "단면 보기"** 토글 버튼. 현재 활성화된 모드가 `$deep-blue` 배경으로 표시.

아이콘을오만 표시하며, 그리드 영역의 폭이 좁아 아이콘을 표시할 수 없게 될 시 아래로 슬라이드 하여 사라진다.

### 5-4. 콘텐츠 본문 — 우측: 페이지 상세 패널 (전체 폭의 약 80~70%)

페이지의 상세 정보를 보여주는 사이드 패널. (Phase 3: text 또는 pdf 내보내기 버튼)

**페이지 메모 영역:**
"페이지 메모" 라벨. contenteditable 텍스트 필드. 예: "전투 씬". 이 필드에 입력한 내용은 Episode 화면의 해당 페이지 행 메모와 양방향 연동된다. 메모 좌측에 **감정 이모티콘 선택기**가 있다 — 작은 원형 버튼을 클릭하면 이모지 피커가 뜨고, 선택한 이모지가 메모 앞에 표시된다.

**페이지 세부 메모 영역:**

페이지 메모 영역 밑의 "**`+ 페이지 메모 추가`**" 를 통해 추가할 수 있다.(예: "A와 B가 XX에서 싸우고 있다...") 호버 시 우측 상단에 X 닫기 버튼이 있다. 페이지 메모와는 디자인에 차이를 두어 부차적인 메모임을 나타낼 수 있도록 한다.
에피소드 세부와 동일한 "#태그"를 사용할 수 있다.

**컷 메모 리스트:**
좌측에는 "컷 메모" 라벨 우측에는 “컷 편집” 버튼.

디폴트는 "**`+ 컷 메모 추가`**" 버튼이 맨 위에 있으며, 컷 메모 추가 시 "**`+ 컷 메모 추가`**" 버튼은 최하단으로 내려간다.

각 컷 메모 행은:

- 드래그 핸들 (컷 메모 순서 변경)
- 컷 번호 (추가 순으로 자동 부여, "1", "2", "3"...)
- 한 줄 메모 텍스트 필드 (예: "건물 전경, 폭발이 일어남")
- 감정 이모티콘 선택기 (작은 원형)
- 삭제 아이콘 (호버 시 표시)

이 컷 메모들은 Cut 화면에서 각 컷에 **드래그 앤 드롭으로 배정**된다. 미배정과 배정됨의 UI가 시각적으로 구분된다 — 미배정은 좌측 보더가 `$amber` 점선, 배정됨은 `$deep-blue` 실선.

## 6. Cut 화면 (컷)

Page 화면에서 특정 페이지 카드를 더블클릭??하면 이 화면으로 진입한다.

### 6-1. 상단 헤더

높이 56px. `$white` 배경.

**좌측:** 에피소드 타이틀 + 현재 페이지 번호가 표시된다. 양면 뷰일 경우 두 페이지가 표시된다. 예: "3화 이웃 친구들 1p, 2p". "3화" `$ocean`, "이웃 친구들" `$ink` 18px 세미볼드, "1p, 2p" `$ink-ghost`. 아래에 **페이지 메모 미리보기**가 `$ink-ghost` 12px 한 줄로 표시된다. 컷 작업 중 페이지의 주요 내용을 복기할 수 있다.

**우측:**

1. **이전 페이지 / 다음 페이지 버튼** — ◁ / ▷ 아이콘 고스트 버튼. 양면 뷰에서는 2페이지 단위로 이동.
2. **전체 페이지 미리보기 버튼 —** 클릭 시 스크롤 위치는 현재 편집 중인 화면인 채로, 위 아래로 전체 페이지가 표시된다. 무한 스크롤 방식처럼 스크롤 시마다 api 호출을 하는 방식으로 불러온다. 다른 페이지는 컷 메모, 컷 세부 메모 이동 외에는 조작할 수 없다. 이 상태에서 컷 메모, 컷 세부 메모를 다른 페이지로 이동할 수 있다. 이 모드일 경우, 현재 페이지가 편집중인 페이지라는 것을 보여주기 위해 편집중인 페이지에는 `$ocean`로 border가 생기고, 나머지 페이지는 배경색이 `$paper`로 하는 등 disabled 같은 느낌을 준다.(Phase 1: 구현x, Phase 2: 미리보기 구현, Phase 3: 크로스 페이지 드래그 구현)
3. **"`메모 패널`"** — ⋯ 아이콘 고스트 버튼.

### 6-2. 콘텐츠 본문 — 중앙 캔버스

본문 영역 전체가 `#e8e6e0`(편집 보드 색상, `$border`보다 약간 어두운)의 배경이다. 이 위에 `$white` 배경의 **B4 규격 페이지**가 중앙에 놓인다.

**단면 뷰:** 하나의 B4 페이지(aspect-ratio 238:324)가 중앙에 표시된다.
**양면 뷰:** 두 개의 B4 페이지가 좌우로 나란히 붙어 하나의 스프레드를 이룬다.

**가이드라인:**
페이지 위에 가이드라인이 오버레이된다.

- **제판선(Bleed Line, H324mm × W238mm):** 페이지의 최외곽. 검정색(`$ink`) 점선.
- **내곽선(Inner Frame, H270mm × W180mm):** 컷이 배치되는 실제 작업 영역. 컷 분할 전에는 파란색(`$ocean`) 실선, 컷이 하나라도 분할된 이후에는 검정색(`$ink`) 실선.

**컷 표시:**
초기 상태에서는 내곽선 전체가 하나의 컷 영역이다(데이터상 컷이 없는 상태). 컷이 분할되면, 각 컷 영역은 내곽선 기준 % 좌표로 배치된다. 각 컷의 경계선은 `$ink` 1px 실선이다.

**컷 내부 표시:**
각 컷 영역 안에는:

- 배정된 컷 메모가 있으면 메모의 첫 줄이 `$ink-soft` 작은 텍스트로 표시된다.
- 감정 이모티콘이 있으면 컷 좌측 상단에 작게 표시.
- 하단에 "**`+ 컷 세부 메모 추가`**" 버튼이 있다. 점선 보더에 + 아이콘과 텍스트. 클릭 시 리스트 맨 아래에 새 행이 추가된다.
- 메모가 보여질 수 없는 크기 이하의 컷 내부에서는 메모 및 버튼을 숨기고 우측 사이드 패널에서만 조회 및 추가/수정할 수 있다.

**컷 세부 메모:**

- **대사/생각:** 선택 시 말풍선 모양을 일반 말풍선/생각 말풍선/플래시 말풍선/뾰족 말풍선 아이콘이 있어, 해당 아이콘 중 하나를 선택하면 메모의 모양이 해당 형태로 변경된다. 메모에 캐릭터 정보 연결 시, 메모 좌측에 캐릭터 얼굴 이미지 또는 이름 첫 글자 아이콘이 표시된다. 말풍선 꼬리는 캐릭터 아이콘 방향을 향해있다. 캐릭터 얼굴 이미지 드래그를 통해 좌측 우측 위치를 변경할 수 있으며, 이에 따라 말풍선 위치도 변경된다.(캔버스 내부에 실제 CSS)
- **나레이션:** 선택시 메모 모양을 배경 `$white`, 경계선은 `$ink` 1px 실선, 또는 배경 `$ink` , 경계선은 `$ink` 1px 실선에 글자색 `$white` 중 선택할 수 있다. 메모에 캐릭터 정보 연결 시, 메모 좌측에 캐릭터 얼굴 이미지 또는 이름 첫글자 아이콘이 표시된다.
- **연출 메모:** 기본 메모 형태로, border-radius가 적용된 형태다.

(Phase 1: 말풍선 css 구현x, Phase 2: 말풍선 구현, Phase 3: 캐릭터 아이콘 위치변경 구현)

선택된 컷은 `$deep-blue` 2px 보더로 강조되고, 호버 시 `$ocean` 1px 보더.

**Ctrl+스크롤:** 마우스 포인터 위치 기준으로 캔버스를 확대/축소한다.

### 6-3. 콘텐츠 본문 — 우측 사이드 패널

캔버스 우측에 약 320px 폭의 사이드 패널이 있다. 기본으로 열려 있으며, 접을 수 있다. (Phase 3: text 또는 pdf 내보내기 버튼)

섹션은 A+B, A+B, … 와 같이 한 컷 마다 컷메모 관리, 그 밑으로 해당 컷의 세부메모 와 같은 형식으로 표시된다. 섹션 B에는 아코디언 접기/펼치기가 적용되어 있으며, 기본 접기 설정이나 선택된 컷은 펼치기가 적용된다. 컷 선택 이외에도 아코디언 클릭으로 펼치기가 가능하다.

**섹션 A: 컷 메모 관리**

Page 화면의 컷 메모 리스트와 동일한 구조. 이 리스트의 각 컷 메모 행을 **캔버스의 특정 컷 영역으로 드래그**하여 배정한다. 배정되면 해당 행의 스타일이 변경된다(좌측 보더 `$amber` 점선 → `$deep-blue` 실선). 배정된 메모를 컷 바깥으로 드래그 앤 드롭하면 배정이 해제된다.

미배정 컷 메모와 배정된 컷 메모의 시각적 차이:

- 미배정: `$amber` 좌측 보더 점선, 배경 `$amber-light` 10% 투명도
- 배정됨: `$deep-blue` 좌측 보더 실선, 배경 `$white`

섹션 최하단의 "`+ 컷 메모 추가`" 버튼으로 이 화면에서도 컷 메모를 새로 생성할 수 있다.

**섹션 B: 컷의 세부 메모**

섹션 A에 들여쓰기 되어 작성되는 등의 형식을 통해(다른 방식이어도 됨) A의 하부 항목임을 표시한다.

"`+ 세부 메모 추가`" 버튼을 누르면 아래에 세부 메모 입력 블록이 추가된다. 각 블록은:

- **타입 선택 드롭다운:** "대사/생각" / "나레이션" / "연출 메모". 선택한 타입에 따라 좌측에 색상 바가 달라진다 (대사=`$deep-blue`, 지문=`$ocean`, 연출=`$amber`).
- **캐릭터 선택 (대사, 나레이션 타입일 때):** 드롭다운으로 프로젝트에서 설정한 캐릭터 중 선택. 캐릭터가 설정되지 않았으면 자유 입력 필드.
- **내용 텍스트 영역:** 대사 내용, 지문 내용, 연출 내용을 자유롭게 기록.
- 드래그 핸들로 세부 메모 간 순서 변경 가능.
- 삭제 버튼(호버 시 표시).

### 6-4. 하단 플로팅 툴바

캔버스 하단 중앙에 떠 있는 가로 바. `$white` 배경, shadow-lg, border-radius 12px.

**좌측**:

1. **되돌리기**(↩) / **다시하기**(↪) 버튼. 컷 분할/합치기/삭제/이동 등의 조작을 undo/redo한다.
2. **구분선** (세로 `$concerto` 1px)
3. **컷 분할 모드 토글** (✂ 아이콘). 클릭하면 활성화되어 `$deep-blue` 배경으로 변하고, 마우스 커서가 컷 분할모드 아이콘으로 변경된다. 활성 상태에서 캔버스의 컷 위를 드래그하면 드래그 방향에 따라 수직/수평/대각선으로 컷이 분할된다. 분할 중에는 분할선 프리뷰가 `$amber` 점선으로 실시간 표시된다.
4. **선택 컷 삭제** (🗑 아이콘). 현재 선택된 컷을 삭제한다. 삭제된 컷의 영역은 인접한 컷이 차지한다.
5. **컷 초기화** (↺ 아이콘). 클릭 시 **토스트 알림**이 뜬다: "컷 분할 작업을 초기화하시겠습니까?" + "□ 이후 표시하지 않기" 체크박스 + "취소" / "초기화" 버튼. 초기화하면 모든 분할이 사라지고 전체 1컷 상태로 돌아간다. 단, 컷별 세부 메모는 컷 메모 하위로 보존되고, 모든 컷 메모가 미배정 UI로 돌아간다.

**우측:**

- **확대**(🔍+) / **화면 맞춤**(⛶) / **축소**(🔍-) 아이콘. 화면 맞춤은 캔버스를 화면에 딱 맞게 리셋.
- 줌 레벨 표시 (예: "150%").

### 6-5. 컷 크기 조절 인터랙션

컷이 분할된 후, 컷 경계선 위에 마우스를 올리면 경계선이 `$deep-blue`로 하이라이트되고, 커서가 col-resize 또는 row-resize로 변한다. 경계선을 드래그하면 인접한 두 컷의 크기가 실시간으로 변한다. 최소 크기 제한이 있어 컷이 10% 미만으로 줄어들지 않는다.

### 6-6. 사용 가능한 키보드 단축키

undo(Ctrl+Z), redo(Ctrl+Shift+Z), 확대(ctrl + +/PAGE UP), 축소(ctrl + -/PAGE DOWN), 삭제(Delete)

## 7. 공통 UI 요소 상세

### 7-1. 상태 뱃지

에피소드 상태를 나타내는 3종 뱃지:

- **예정(draft):** `$glacial` 배경, `$ink-soft` 텍스트, border-radius 4px, 좌측에 ○ 빈 원 아이콘.
- **진행중(inProgress):** `$ocean` 배경, `$white` 텍스트, 좌측에 ◑ 반원 아이콘.
- **완료(done):** `$deep-blue` 배경, `$white` 텍스트, 좌측에 ✓ 체크 아이콘.

### 7-2. 토스트 알림

화면 하단 중앙에서 슬라이드 업 되는 가로 바. `$white` 배경, shadow-lg, border-radius 8px, 좌측에 상태 아이콘(정보=`$ocean` ⓘ, 경고=`$amber` ⚠, 에러=빨강 ✕), 메시지 텍스트, 우측에 액션 버튼(들). 5초 후 자동으로 슬라이드 다운 되어 사라지거나, 사용자가 직접 닫을 수 있다.

### 7-3. 권한별 UI 변화

- **소유자(owner):** 모든 UI 요소 활성.
- **편집자(editor):** 소유자와 거의 동일하나, 프로젝트/에피소드 삭제와 공유 설정 변경 불가. 해당 버튼이 비활성 또는 미노출.
- **댓글 작성자(commentonly):** 메모/세부 메모/컷 메모는 읽기 전용. 별도의 "코멘트 남기기" 기능만 활성(향후 구현).
- **읽기 전용(readonly):** 모든 입력 필드와 버튼이 비활성. 드래그 앤 드롭 불가. 보기 전용 모드.

링크로 접근 시 권한이 없으면, 렌더링 전 시점에 토스트 알림 "접근 권한이 없습니다"가 뜨고, "확인" 클릭 또는 5초 후 이전 페이지로 자동 이동.

### 7-4. 설정 상속 체인 표시

에피소드 개별 설정 모달에서는 각 설정 항목 옆에 "프로젝트 기본값: 24" 같은 회색 힌트가 표시되어, 현재 값이 상속된 것인지 개별 오버라이드된 것인지 구분할 수 있다. 오버라이드된 항목은 "리셋" 버튼이 나타나 클릭하면 상위 설정으로 되돌릴 수 있다.

### 7-5. 사이드바-검색 모달

- **상단:** 큰 검색 입력 창. Enter를 눌러야 검색 결과가 표시된다.
- **하단:** "프로젝트 / 에피소드 / 페이지 / 컷" 4개 탭이 가로로 나열되고, 탭 별로 검색 결과가 리스트로 표시된다. 보여지는 검색 결과는 검색어가 하이라이트 되서 표시된다. 각 결과 행에는 해당 항목의 아이콘, 제목, 소속 경로(예: "고양이와 강아지 > ep.3 > p.05")가 있고, 클릭 시 해당 화면으로 바로 이동한다. 검색 결과가 없을 시 “검색 결과가 없습니다.”라는 텍스트가 중앙에 표시된다. 검색 범위는 내 프로젝트와 공유된 프로젝트 전체이다. 최근 검색어 표시는 없다.

### 7-6. 메모 패널: ⋯(가로 점 3개) 아이콘의 고스트 버튼

- **상단:** 좌측에 “메모”, 우측에 핀 아이콘과 검색 버튼, X 닫기 버튼. (Phase 3: text 또는 pdf 내보내기 버튼)
  - 검색 버튼을 누를 시 하단에 검색 입력 창이 표시되고, 엔터를 누를 시 해당 프로젝트 내에서 탭 별로 검색 결과가 표시된다. 보여지는 검색 결과는 검색어가 하이라이트 되서 표시된다.
  - 핀 아이콘을 누를 시 메모 패널은 닫기 버튼을 누르거나 핀 아이콘을 해제하지 않는 이상 모달이 계속 표시된 상태이다.
- **하단:** "프로젝트 / 에피소드 / 페이지 / 컷" 4개 탭이 가로로 나열되고, 탭 별로 현재 접속 중인 프로젝트 내부의 모든 메모를 볼 수 있다. 메모는 순서대로 표시되며, 세부 메모는 아코디언 닫기 처리 되어있다. 단, 검색 결과가 세부 메모에 있을 경우 펼쳐진 상태로 보여진다. 현재 작업중인 페이지에 스크롤이 위치해있다.(예: 에피소드 8화를 수정 중이었으면 메모 패널을 열 시, 에피소드 탭을 열었을 때 에피소드 8화에 스크롤이 위치함)

### 7-7. **에러/빈 상태(Empty State)**

- **프로젝트가 없는 책장:** 사이드바에서는 클릭 시 좌측 삼각형▼/▶은 변동되나, 프로젝트 수준의 들여쓰기가 되어 있고 “프로젝트가 없습니다.” 라고 표시되어 있다. 콘텐츠 본문에서는 섹션 헤더 영역에 호버해도 다른 섹션과는 다른 disabled스러운 식으로 호버 스타일이 나타나고, 클릭해도 반응이 없다. 섹션 헤더 자체도 disabled스럽게 처리되어 있다.
- **에피소드가 없는 프로젝트:** 사이드바에서는 클릭 시 좌측 삼각형▼/▶은 변동되나, 에피소드 수준의 들여쓰기가 되어 있고 “에피소드가 없습니다.” 라고 표시되어 있다. 에피소드 목록 영역에는 "**`+ 에피소드 추가`**" 행만 보인다.
- **페이지가 없는 에피소드:** 사이드바에서는 클릭 시 좌측 삼각형▼/▶은 변동되나, 페이지 수준의 들여쓰기가 되어 있고 “페이지가 없습니다.” 라고 표시되어 있다.페이지별 메모 리스트에는 "`+ 페이지 추가`" 행이만 보인다.

### 7-8. 삭제 확인 플로우

- **프로젝트/에피소드/페이지/컷 에셋 삭제:** "정말 삭제하시겠습니까?" 같은 확인 단계를 사용. 아래에 “다음부터 띄우지 않기” 체크박스가 있다.
- **메모 삭제:** 바로 삭제 후 undo 토스트를 띄운다.

## 8. 라우팅 구조

| 경로                                                  | 화면      | 비고                            |
| ----------------------------------------------------- | --------- | ------------------------------- |
| `/`                                                   | Bookshelf | 루트 = 책장                     |
| `/project/:projectId`                                 | Project   | 프로젝트 상세 + 에피소드 리스트 |
| `/project/:projectId/episode`                         | Episode   | 에피소드 네비게이터 + 상세      |
| `/project/:projectId/episode/:episodeId/page`         | Page      | 페이지 그리드 + 상세            |
| `/project/:projectId/episode/:episodeId/page/:pageId` | Cut       | 컷 캔버스 + 사이드 패널         |

사이드바 트리와 URL은 항상 동기화된다. 어느 화면에서든 사이드바 트리의 해당 경로 노드가 하이라이트 되며, 트리 노드를 클릭하면 해당 URL로 이동한다.

## 9. 데이터 흐름 요약

- 모든 화면의 메모는 단일 Memo 테이블(`parentId`, `parentType`, `role`)에 정규화되어 저장된다. "에피소드 화면에서 작성한 페이지 메모"와 "페이지 화면에서 보이는 페이지 메모"는 동일한 Memo 레코드를 참조하므로 양방향 연동이 보장된다. 컷 메모도 마찬가지로, Page에서 작성한 컷 메모와 Cut에서 배정/편집하는 컷 메모는 같은 데이터다.
- 설정은 Episode > Project > User > Default 순으로 상속되며, null인 필드는 상위 값을 따르고, 값이 있으면 오버라이드한다.

## 10. 기술된 내용 외 차후 구현 예정내용

- **헙업용 코멘트 시스템:** 각 메모에 편집자나 어시스턴트가 특정 좌표에 피드백 코멘트를 남길 수 있는 기능.
- **제안모드:** 즉시 수정하는 것이 아닌, 편집자가 확인할 수 있도록 메모를 수정 제안하는 기능.
- **Infrastructure:** 서버 저장 및 로그인 기능 도입 예정 (JSON 구조는 서버와 1:1 대응 필수). 서버 저장 시 서버 저장이 되었음을 확인시켜주는 란이 필요하며, 서버 저장이 되지 않았는데 페이지를 떠날 때에는 확인 알람이 떠야 한다.
- **Access Control:** 다중 계정 공유 및 권한 분리(열람/수정), 회원 등급별(비회원/무료) 접근 제한 로직 고려.
- **Scalability:** 일본어, 영어 등 다국어 대응을 고려한 i18n 구조 설계.
- **Mobile App:** Flutter 기반의 모바일 앱 대응 예정 (Cross-platform 데이터 호환성 고려 필수)
- **text 또는 pdf 내보내기:** 전체 또는 일부 지정(예: 페이지 2부터 8까지)하여 text 또는 pdf로 내보내는 기능.
